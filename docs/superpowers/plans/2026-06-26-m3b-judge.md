# M3b — AI Judge + Review Queue Implementation Plan

> **For agentic workers:** This project runs the Claude↔Codex 老流程 (see `AGENTS.md`): **Codex implements** each task via `codex:codex-rescue` (`--write`); **Claude runs tests + reviews + commits** per task and at a milestone gate (绿测试≠Done). Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Score a candidate's answer to a practice question on five dimensions (1–5) **using only the evidence bound to that question**, plus a diagnosis, an English rewrite, and follow-up questions; persist the result onto the `practice_run`; and push any dimension scored ≤2 into the `review_item` queue.

**Architecture:** Mirror M3a's generate→persist shape. `judgeAnswer` is the only LLM call; it receives the question + **only the `practice_run`'s bound evidence** (`getPracticeRunBoundEvidence` from M3a) + the answer (`answer_text` or `transcript`). A question/score is a *judgment*, not a deterministically verifiable factual claim, so there is **no M0d-style validator gate** here — the red-line "judge only on bound evidence, not model prior" is enforced structurally (the prompt is given only the bound evidence + an explicit "no outside knowledge" instruction) rather than by a deterministic checker. Persistence is one transaction: update `practice_run` + replace this run's `review_item` rows (idempotent re-judging under the `unique(practice_run_id, dimension)` constraint).

**Tech Stack:** TypeScript (strict, `noUncheckedIndexedAccess`), zod, Vercel AI SDK v7 via the existing `LlmClient`, better-sqlite3, vitest. LLM only through `lib/llm` (red line #2); judge sees only bound evidence (red line #1); `MockLlmClient` in tests (red line #5).

> **Revised after Codex design-review round 1** (CONDITIONAL GO → fixes integrated): P1 the red-line-#1 guarantee was not actually *tested* (the mock ignores the prompt) → Task 4 adds a prompt-capturing client asserting the bound evidence is present, an unbound evidence_unit is absent, and the answer text is present. P2 Task 3 test now asserts `rewrite` persistence. P2 `applyJudgeResult` validates each score is an integer 1-5 before any write (defense-in-depth: `review_item.score` has no DB CHECK and the migration is already shipped).

```ts
  boundary: scoreSchema,

});
export const JudgeResultSchema = z.object({
  scores: JudgeScoresSchema,
  diagnosis: z.string().min(1),
  rewrite: z.string().min(1),
  follow_ups: z.array(z.string()), // may be empty

export type JudgeScores = z.infer<typeof JudgeScoresSchema>;
export type JudgeResult = z.infer<typeof JudgeResultSchema>;
```

- The five score keys are exactly the `review_item.dimension` CHECK values (`evidence|clarity|completeness|boundary|delivery`), so persisting low scores needs no mapping.
- `scoreSchema` is `int().min(1).max(5)` — the model cannot emit 0/6/non-integer; the repository additionally treats any missing dimension as an error (defensive).

```ts
// src/lib/llm/judge.ts
import { z } from "zod";
import type { LlmClient } from "./types";

export const DIMENSIONS = ["evidence", "clarity", "completeness", "boundary", "delivery"] as const;
export type Dimension = (typeof DIMENSIONS)[number];

const scoreSchema = z.number().int().min(1).max(5);
export const JudgeScoresSchema = z.object({
  evidence: scoreSchema,
  clarity: scoreSchema,
  completeness: scoreSchema,
  boundary: scoreSchema,
  delivery: scoreSchema,
});
export const JudgeResultSchema = z.object({
  scores: JudgeScoresSchema,
  diagnosis: z.string().min(1),
  rewrite: z.string().min(1),
  follow_ups: z.array(z.string()),
});
export type JudgeScores = z.infer<typeof JudgeScoresSchema>;
export type JudgeResult = z.infer<typeof JudgeResultSchema>;

export function buildJudgePrompt(args: {
  question: string;
  evidence: { id: string; text: string }[];
  answer: string;
}): string {
  const ev = args.evidence.map((e) => `[${e.id}] ${e.text}`).join("\n");
  return [
    `You are a viva (thesis defence) examiner scoring a candidate's spoken/written answer.`,
    `Judge ONLY against the EVIDENCE below. Do not use outside knowledge. If the answer asserts something not supported by this evidence, the "evidence" score must be low.`,
    `Score each dimension as an integer 1-5 (1 worst, 5 best):`,
    `- evidence: how well the answer is supported by and consistent with the provided evidence`,
    `- clarity: how clear and well-structured the answer is`,
    `- completeness: how fully it addresses the question`,
    `- boundary: awareness of scope, assumptions, and limitations`,
    `- delivery: quality of English expression`,
    `Also return: a brief diagnosis, an improved English rewrite of the answer grounded in the evidence, and 0-3 follow-up questions.`,
    ``,
    `QUESTION: ${args.question}`,
    ``,
    `EVIDENCE (id: text):`,
    ev,
    ``,
    `CANDIDATE ANSWER: ${args.answer}`,
  ].join("\n");
}

export async function judgeAnswer(
  client: LlmClient,
  args: { thesisId: string; question: string; evidence: { id: string; text: string }[]; answer: string },
): Promise<JudgeResult> {
  return client.generateObject({
    role: "hard",
    purpose: "judge",
    schema: JudgeResultSchema,
    prompt: buildJudgePrompt(args),
    thesisId: args.thesisId,
  });
}
```

```ts
export const REVIEW_DIMENSIONS = ["evidence", "clarity", "completeness", "boundary", "delivery"] as const;
export const REVIEW_SCORE_THRESHOLD = 2;

export function getPracticeRunForJudge(
  db: DB,
  practiceRunId: string,
): { thesisId: string; question: string; answerText: string | null; transcript: string | null } | undefined {
  const row = db
    .prepare("SELECT thesis_id, question, answer_text, transcript FROM practice_run WHERE id=?")
    .get(practiceRunId) as { thesis_id: string; question: string; answer_text: string | null; transcript: string | null } | undefined;
  if (!row) return undefined;
  return { thesisId: row.thesis_id, question: row.question, answerText: row.answer_text, transcript: row.transcript };
}

/** Persist a judge result onto the practice_run AND refresh its review queue, atomically.
 *  Returns the dimensions that were pushed to review (score <= REVIEW_SCORE_THRESHOLD). */
export function applyJudgeResult(
  db: DB,
  input: {
    practiceRunId: string;
    thesisId: string;
    scores: Record<string, number>;
    diagnosis: string;
    rewrite: string;
    followUps: string[];
  },
): string[] {
  // Validate every dimension up front — defense-in-depth: review_item.score has no 1-5
  // DB CHECK (migration already shipped), so a direct caller must not be able to persist
  // an out-of-range/non-integer score. Fail before any write.
  const reviewed: { dim: string; score: number }[] = [];
  for (const dim of REVIEW_DIMENSIONS) {
    const score = input.scores[dim];
    if (score === undefined) throw new Error(`missing score for dimension: ${dim}`);
    if (!Number.isInteger(score) || score < 1 || score > 5) {
      throw new Error(`score for ${dim} must be an integer 1-5, got: ${score}`);
    }
    if (score <= REVIEW_SCORE_THRESHOLD) reviewed.push({ dim, score });
  }
  const tx = db.transaction(() => {
    db.prepare(
      `UPDATE practice_run SET scores=@scores, diagnosis=@diagnosis, rewrite=@rewrite, follow_ups=@follow_ups
        WHERE id=@id`,
    ).run({
      id: input.practiceRunId,
      scores: JSON.stringify(input.scores),
      diagnosis: input.diagnosis,
      rewrite: input.rewrite,
      follow_ups: JSON.stringify(input.followUps),
    });
    db.prepare("DELETE FROM review_item WHERE practice_run_id=?").run(input.practiceRunId);
    const ins = db.prepare(
      "INSERT INTO review_item (id, thesis_id, practice_run_id, dimension, score, reason) VALUES (?,?,?,?,?,?)",
    );
    for (const r of reviewed) {
      ins.run(randomUUID(), input.thesisId, input.practiceRunId, r.dim, r.score, input.diagnosis);
    }
  });
  tx();
  return reviewed.map((r) => r.dim);
}
```

```ts
import { getPracticeRunForJudge, getPracticeRunBoundEvidence, applyJudgeResult } from "../../db/repository";

export async function runJudge(
  db: DB,
  client: LlmClient,
  practiceRunId: string,
): Promise<{ scores: JudgeScores; reviewDimensions: string[] }> {
  const run = getPracticeRunForJudge(db, practiceRunId);
  if (!run) throw new Error(`practice_run not found: ${practiceRunId}`);

  const answer = (run.answerText ?? "").trim() || (run.transcript ?? "").trim();
  if (!answer) throw new Error("practice_run has no answer (or transcript) to judge");

  const evidence = getPracticeRunBoundEvidence(db, practiceRunId);
  if (evidence.length === 0) throw new Error("practice_run has no bound evidence to judge against");

  const result = await judgeAnswer(client, { thesisId: run.thesisId, question: run.question, evidence, answer });

  const reviewDimensions = applyJudgeResult(db, {
    practiceRunId,
    thesisId: run.thesisId,
    scores: result.scores,
    diagnosis: result.diagnosis,
    rewrite: result.rewrite,
    followUps: result.follow_ups,
  });

  return { scores: result.scores, reviewDimensions };
}
```

## Red-line / safety checklist (verify before the milestone gate)

1. **Judge on bound evidence only (red line #1):** `runJudge` feeds `judgeAnswer` exactly `getPracticeRunBoundEvidence(practiceRunId)` — never the whole thesis — and the prompt says "do not use outside knowledge; if the answer asserts something not in the evidence, evidence score must be low." A run with zero bound evidence is rejected (cannot judge against nothing). There is intentionally **no** deterministic validator: a 1–5 score is a judgment, not a provable fact (contrast M0d/M2, which gate *factual content*). This is a deliberate, documented boundary.
2. **No partial writes:** `applyJudgeResult` updates `practice_run` and refreshes `review_item` in one transaction; re-judging replaces this run's review rows (idempotent under `unique(practice_run_id, dimension)`). The LLM call happens before the transaction, so an LLM failure persists nothing.
3. **LLM only via `lib/llm` (red line #2):** the sole model call is `judgeAnswer` → `client.generateObject` (`role: "hard"`, `purpose: "judge"`); no provider SDK, no hardcoded model.
4. **Graceful degrade (red line #4):** a disabled client rejects; nothing is persisted; the practice transcript/answer remains intact for a later judge attempt.
5. **Tests use `MockLlmClient` (red line #5):** no live calls.
